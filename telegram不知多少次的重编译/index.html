    Telegram-iOS builds its video calls upon webrtc-ios, which is a gigantic module imported this year. Itâ€™s supposed to be built to a framework by buck before generating workspace files. Unfortunately, the code in the official repo doesnâ€™t compile with Xcode 12.x and doesnâ€™t support thex86_64 target.

    The first change is on a python script find_sdk.py which is a part of webrtc-ios build system. It helps to locate the SDK path in Xcode.app. The original regular expression ^MacOSX(10.\d+).sdk$ doesnâ€™t work against Xcode 12.x, as the SDK pathname has been changed to Contents/Developer/Platforms/MacOSX.platform/Developer/SDKs/MacOSX11.0.sdk.
