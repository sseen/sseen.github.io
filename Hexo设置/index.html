Next 主题自定义样式

发现 Next主题 真是好用又好看，然后换掉了之前一直用的 icarus。
而且样式自定义的方式很合理，看 Next主题 的 _config.yml，搜索 custom 字样就能找到方式。
目前的8.0应该是这样的：
首先，Next主题 的 _config.yml 文件里打开注释的改成这样；

style: source/_data/styles.styl

其次，在 Hexo 的 source 目录下新建 _data/styles.styl 文件；
然后，在这个文件里添加你想要的样式，具体的 css 选择器目标，直接看生成的静态文件页面就可以了。
例如，想把文章里 tag 的背景去掉。

.posts-expand .post-tags a {
    background: transparent;
    }

_post源目录文章归类文件夹

断续记录了这么些次，最开始的文章放在 _posts 下有些乱了；虽然后面发现了这个命令可以方便存相应的年份下，方便管理

hexo new -p 2020/"Hexo设置" "Hexo设置"
hexo new -p 2025/"" ""
#  本地预览
hexo clean && hexo g && hexo s

但是以前的文章要怎么操作呢？

这里说了这种情况，实际自己操作下，发现问题并不复杂，使用子文件夹管理 Hexo 文章且不改变文章永久链接。

首先，如果嫌弃 Hexo 生成的文章链接太复杂（因为加了日期）可以在 _config.yaml 中把 permalink 改成 permalink: :name/；
其次，直接在 _posts 下新建年份目录如 2016，然后把属于这个年份的文章都加进去就可以了。
