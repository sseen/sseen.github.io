iMac 5k 15 系统重装出现问号文件夹问题

Posted on 2020-09-30 Edited on 2025-11-01 Views:

自己15 iMac 5k突然有一天发现，重启以后系统回到了前两天的状态，这两天的内容全部丢失的情况。当时装了big sur beta6，然后想着抹掉重装catalina(10.15.6)，然后就出现开机启动出现闪烁的问号文件夹。

🌤第1次尝试

command + option + R 抹盘重装安装catalina(10.15.6)就不行了。
command + R 还能装15年的 el capitan(10.11.6)。

那既然能装 el capitan(10.11.6)，装上后直接升级总可以的吧，然后安装更新 capitan(10.11.6)，启动一样的闪烁 “folder?”，wtf?

那好，一级级的升级总不会有问题了吧，sierra(10.12) 先来？一片曙光啊，竟然可以，心里稳了。继续 high sierra(10.13) 吧？咦，熟悉的启动闪烁又出现了，崩了。

然后试了更种组合的更新方法，都宣告失败。无奈之下，只能网上预约苹果官方店查查看，当然是先给客服聊了半天，也远程支持了，包括检测硬件命令都试过了，一切正常，还是不知道原因。

11号送修了，过了三四天，守着电话终于响了，说试了各种方法，包括不限于互换正常硬盘安装，硬盘没问题，排除硬盘问题，那就是硬件主板可能坏了，修一下4000，直接换新的，我一想太贵了吧。那说帮我在别的机器上装个新的系统，然后放回，就先用着，也还能用几年，我一想也只能这样了。

兴奋的开车取回家，赶紧试试新系统吧，谁知道一开机帮我装的 mojave(10.14) 我的 catalina(10.15.6) 呢，算了，免费帮我折腾了几天就算了。然后新建个文件，一顿日常操作，重启看看吧。一启动傻眼了，回到了最开始的状态，所有的资料都不见了，又到了 mojave(10.14) 开机页面。虽然自己还能一直用 sierra(10.12)，但是太老了，很多软件可能都不支持了，包括自己写的软件哈哈，太讽刺了。但是实在没办法，网上搜了一通，也说iMac 5k 15有问题，什么boot rom还是smc版本不能升级了之类的，我宣告放弃了。

🌧第二次尝试

持续用了半个多月的 sierra(10.12)，没有兴奋的感觉，都是能用就好的将就。终于在前天也就是28号，又想起来搜下看到底是什么问题。找到了这篇文章【疑难杂症】解决苹果MacOS升级提示“验证固件时发生错误”，无论如何都无法升级更新，主板Boot Rom、SMC版本号更新！，好像发现了新大陆一样，跟我的情况简直如出一辙，激动之余赶紧动手拿出媳妇的移动硬盘，想在外接的硬盘上安装升级 high sierra(10.13)，这样机器上的boot rom就可以升级了，然后就可以用原硬盘正常升级了。

第二次开始行动。惊喜发现真的可以升级到 high sierra(10.13)，这是之前原装硬盘办不到的事情，又燃起了极大的信心。然后到了凌晨1点多，看着半泽2最终章，发现外接硬盘升级了 high sierra(10.13) 后 boot rom版本号并没有变化，回到原装系统 sierra(10.12) 依然无法正常升级，残念です。

然后第二天6点多就起来了，继续搞机器。漫天的找 “High Sierra Update Firmware Error - FIX”, “MacOS High Sierra: An error occurred while verifying firmware”, “Catalina bricked my 2015 MacBook Pro. What to do now?”, “Boot ROM Version eines iMac 17,1 von Version 170.0.0.0.0 auf 176.0.0.0.0 aktualisieren?”, Some Macs don’t update their firmware when they should 等等，想要寻找答案…

🌈最终

功夫不负有心人。看到这个文章送修Mac前 万能的两个自救方法：重置 NVRAM（PRAM）与 重置 SMC，其实也内心先否定了一下，因为iMac重置SMC我第一天就操作了，没有用。但是精疲力尽之余，死马当活马医，最后再试一次鬼使神差的重置了 NVRAM（PRAM），然后直接原装机器升级 high sierra(10.13) 竟然成功了。我一鼓作气 mojave(10.14), catalina(10.15.6), big sur beta8, 都顺利的装好了。苍天啊，这么简单的一个操作，经历了快一个月…

感恩，感谢，互联网…
